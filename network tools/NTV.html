NetworkTrafficView

Definizione

NetworkTrafficView è uno strumento di monitoraggio della
rete che cattura i pacchetti che passano attraverso
l'adattatore di rete e visualizza statistiche generali
sul traffico di rete. Le statistiche dei pacchetti sono
raggruppate per tipo Ethernet, protocollo IP, indirizzi
di origine/destinazione e porte di origine/destinazione.
Per ogni riga delle statistiche vengono visualizzate le
seguenti informazioni: Tipo Ethernet (IPv4, IPv6, ARP),
Protocollo IP (TCP, UDP, ICMP), Indirizzo sorgente,
Indirizzo di destinazione, Porta sorgente, Porta
destinazione, Nome servizio (http, ftp, e così via),
conteggio pacchetti, dimensione totale pacchetti,
dimensione totale dati, velocità dati, velocità massima
dati, dimensione media pacchetto, ora primo/ultimo
pacchetto, durata e ID/nome processo (per connessioni TCP)

Requisiti di sistema

Questa utility funziona su qualsiasi versione di Windows,
a partire da Windows 2000 e fino a Windows 10, inclusi i
sistemi a 64 bit. Per utilizzare NetworkTrafficView è
necessario uno dei seguenti driver di acquisizione:
Driver di acquisizione Npcap Npcap è un driver di acquisizione
open source basato sulla libreria WinPcap fuori produzione
che consente di acquisire pacchetti di rete su qualsiasi
versione di Windows. È possibile scaricare e installare
il driver Npcap da questa pagina Web .
Driver Microsoft Network Monitor versione 2.x
(solo per Windows 2000/XP/2003): Microsoft fornisce un driver di
acquisizione gratuito in Windows 2000/XP/2003 che può
essere utilizzato da NetworkTrafficView, ma questo
driver non è installato per impostazione predefinita e
è necessario installarlo manualmente, utilizzando una
delle seguenti opzioni:
Opzione 1: installarlo dal CD-ROM di Windows 2000/XP
secondo le istruzioni nel sito Web Microsoft
Opzione 2 (solo XP): scaricare e installare gli strumenti
di supporto di Windows XP Service Pack 2 . Uno degli
strumenti in questo pacchetto è netcap.exe. Quando
esegui questo strumento per la prima volta, il driver del
monitor di rete verrà installato automaticamente sul tuo
sistema.
Driver Microsoft Network Monitor versione 3.x: Microsoft
fornisce una nuova versione del driver Microsoft
Network Monitor (3.x) supportata anche in Windows 7/Vista/2008.
La nuova versione di Microsoft Network Monitor (3.x) è
disponibile per il download dal sito Web Microsoft .
Puoi anche provare a utilizzare NetworkTrafficView senza
installare alcun driver, utilizzando il metodo 'Raw
Sockets'. Sfortunatamente, il metodo Raw Sockets presenta
molti problemi:
Non funziona su tutti i sistemi Windows, a seconda della
versione di Windows, del service pack e degli
aggiornamenti installati sul sistema. Su alcuni sistemi
Raw Sockets funziona solo parzialmente e cattura solo
i pacchetti in entrata. Su alcuni altri sistemi, non
funziona affatto.
Sui sistemi in cui il metodo 'Raw Sockets' funziona
correttamente, può acquisire solo pacchetti TCP/UDP IPv4.
Non può catturare altri tipi di pacchetti, come gli altri
driver di cattura.
Su Windows 7 con UAC attivato, il metodo "Raw Sockets"
funziona solo quando si esegue NetworkTrafficView con
"Esegui come amministratore".

Utilizzando AdapterWatch

Ad eccezione di un driver di acquisizione necessario per acquisire
i pacchetti di rete, NetworkTrafficView non richiede alcun processo
di installazione o file dll aggiuntivi. Per iniziare a usarlo,
esegui semplicemente il file eseguibile - NetworkTrafficView.exe
Dopo aver eseguito NetworkTrafficView per la prima volta,
sullo schermo viene visualizzata la finestra "Opzioni di acquisizione"
e ti viene richiesto di scegliere il metodo di acquisizione e l'adattatore
di rete desiderato. La prossima volta che utilizzerai NetworkTrafficView,
inizierà automaticamente ad acquisire i pacchetti con il metodo di
acquisizione e l'adattatore di rete selezionati in precedenza. Puoi sempre
modificare nuovamente le "Opzioni di acquisizione" premendo F9.
Dopo aver scelto il metodo di acquisizione e l'adattatore di rete,
NetworkTrafficView inizia a visualizzare il traffico di rete corrente,
raggruppato per tipo Ethernet, protocollo IP, indirizzi di origine/destinazione e
porte di origine/destinazione.
È possibile premere F6 per interrompere l'acquisizione del traffico di
rete, F5 per riavviarla o Ctrl+X per cancellare le statistiche attuali
del traffico di rete.
